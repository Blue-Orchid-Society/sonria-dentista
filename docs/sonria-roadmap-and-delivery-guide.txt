SONRIA DENTISTA WEBSITE ROADMAP AND DELIVERY GUIDE

Purpose

We are rebuilding and improving the Sonria Dentista website so it becomes a real patient lead engine, not just a brochure website.

The new website should use the current live site as the first source of truth for business information, then improve the structure, design, conversion flow, SEO, analytics, and lead capture.

The goal is to help Sonria compete locally against larger dental chains by emphasizing private-practice trust, bilingual care, transparent pricing, real patient reviews, convenient locations, and fast appointment requests.

Delivery Workflow

We should keep the work organized by branch and by concern.

Main branch:
Stable production-ready code only.

Planning branch:
ai/site-roadmap-deployment-guide
Used for roadmap, planning, and documentation work.

Feature branches:
One concern per branch.
Examples:
seo/location-content-update
seo/service-page-expansion
dev/contact-form-lead-routing
design/homepage-conversion-refresh
dev/analytics-conversion-tracking

Pull request rule:
Each pull request should focus on one concern.
Do not combine design changes, SEO content rewrites, analytics setup, and form routing in one large PR unless there is a strong reason.

Deployment rule:
Every branch should be deployable separately as a Vercel preview deployment.
We should use preview deployments for review before merging to main.
The production domain should only point to main after QA and approval.

Coding Guide

Use the existing Next.js structure.

Content belongs in:
data/site.en.json
data/site.es.json

Components belong in:
components/sections/

Routes belong in:
app/[locale]/

API routes belong in:
app/api/

Rules:
Keep English and Spanish content shapes aligned.
If a field is added to English, add the matching field to Spanish in the same PR.
Do not add a CMS.
Do not add heavy animation libraries.
Do not add Three.js.
Do not commit API keys.
Do not commit .env.local.
Do not change the Vercel scope.
Do not make large unrelated refactors.
Prefer small, reviewable changes.


Design Guide

The design should feel like a trusted neighborhood family dentist.

Design direction:
Warm
Friendly
Professional
Bilingual
Clear
Practical
Conversion-focused

Avoid:
Overly corporate design
Aggressive sales language
Generic stock-photo feel
Too many decorative cards
Heavy animations
Hard-to-read text
Hidden CTAs

Important design priorities:
Mobile call button should be easy to find
Appointment request should be easy to start
Location information should be very clear
Pricing and specials should be easy to scan
Reviews should appear early enough to build trust
Real photos should replace generic placeholders where possible
